### Content Delivered

Comprehensive analysis of how the Copilot SDK supports or constrains import/export of agents and skills.

**7 Section Analysis:**

1. **Session Model Portability** — Sessions are ephemeral. Portable: charters, tools, MCP configs. Not portable: session state, tool handlers, session-scoped hooks.

2. **Tool Registration & Conflicts** — SDK has no per-agent tool filtering/namespacing. Failure mode: imported Agent A's tool collides with Agent B's identical tool name.

3. **Authentication & Rate Limiting** — GitHub API: 5K req/hr authenticated. Constraint: bulk imports without auth hit rate limits. Solution: pre-flight validation + exponential backoff.

4. **Versioning & Breaking Changes** — SDK is Technical Preview (v0.1.x). No SemVer guarantee. Scenario: agent built on v0.1.8, consumer has v0.2.x → unknown fields silently dropped.

5. **Platform Constraints** — File size >100KB, path separators (Windows vs Unix), non-UTF8 encoding, MCP localhost URLs not portable.

6. **SDK Help vs. Squad Must Build:**
   - SDK provides: CustomAgentConfig serialization
   - Squad must build: tool namespacing, MCP validation, version adapters, transaction safety

7. **Customer Impact Matrix** — 8 failure modes customers will hit during import.
