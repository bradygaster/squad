# Advanced Squad Session — 500-Level Engineering Deck

This package is a self-contained Build-style technical presentation for the advanced Squad session. The narrative is problem-first and evidence-led: context pressure, memory governance, two-layer state backend mechanics, prompt/context slimming, presets, cross-squad contracts, and squads spawning squads.

## Scope

The deck focuses on the requested advanced themes:

- Context budget pressure, using issue #1037 and Tamir's agent-memory blog as the opening problem.
- Squad memory layers: decisions, inbox, histories, skills, governed memory, git notes, and orphan state.
- Governed memory model from PR #1145: classes, load guidance, provider boundaries, CLI proof, and audit trail.
- Real two-layer backend proof from a throwaway demo repo: `squad-state`, git notes, promotion, archive, and logs.
- Context-size reductions beyond memory, including split/lazy coordinator references from #1036 and PR #1035.
- Presets as reusable squad-definition patterns.
- Cross-squad communication: manifests, `squad discover`, `squad delegate`, and issue-based handoff.
- Squads spawning squads, based on Tamir's fanout/Holodeck posts and PR #1022's E2E template-testing skill.

## Regenerating memory evidence

From the repository root, build the local CLI first:

```powershell
$env:SKIP_BUILD_BUMP='1'
npm run build -w packages/squad-sdk
npm run build -w packages/squad-cli
```

Then run the commands in `prompts/04-memory-cli-proof.md` from `docs/advanced-squad-session/demo-root`. Checked-in output files are sanitized to replace local absolute paths with `<demo-root>` and `<repo>`.

## Regenerating PowerPoint

`advanced-squad-session.pptx` is regenerated by rendering each slide in `deck.html` at 1280x720 with Playwright and inserting the images into PowerPoint with `python-pptx`. Run generation from this worktree so `node_modules` and workspace paths resolve.

Validation target:

- `deck.html` script parses.
- Browser DOM reports the expected `.slide` count (`23` for this deck).
- PPTX opens as a valid zip package and contains the same number of slides as `deck.html`.
